# GitHub Pages Deployment Guide

## Overview

This project is configured for easy deployment to GitHub Pages. The build process:
1. Builds client and server bundles with Vite
2. Runs a post-build script (`scripts/gen-index.mjs`) to generate `index.html`
3. Deploys the `dist/client/` directory to GitHub Pages

## Build Outputs

```
dist/
├── client/              # GitHub Pages deployment directory
│   ├── index.html       # Generated by post-build script
│   ├── assets/          # Bundled JavaScript, CSS, and images
│   └── *.jpg            # Wallpaper images
└── server/              # Server-rendered files (not used for GitHub Pages)
```

## Quick Start

### Automatic Deployment (GitHub Actions) ⭐ Recommended

1. Ensure `.github/workflows/deploy.yml` exists (already included)
2. Push to `main` branch
3. GitHub Actions automatically:
   - Builds the app with correct base URL (`/pixel-sanctuary/`)
   - Generates `index.html` from client assets
   - Uploads to GitHub Pages
   - Deploys to `https://shonejj.github.io/pixel-sanctuary/`

No additional setup needed beyond initial repository configuration.

**GitHub Pages Settings:**
- Settings → Pages
- Source: **Deploy from a branch**
- Branch: **gh-pages** (created automatically by the workflow)

### Manual Deployment with gh-pages

```bash
# First time setup
bun add -d gh-pages

# Build and deploy
bun run deploy:pages
```

This runs:
1. `build:pages` - Builds with `/pixel-sanctuary/` base URL + generates index.html
2. `npx gh-pages -d dist/client` - Uploads to gh-pages branch

## Build Process Details

### Build Scripts

| Script | Purpose |
|--------|---------|
| `bun run build` | Build for development (outputs server + client) |
| `bun run build:dev` | Build in development mode |
| `bun run build:pages` | Build for GitHub Pages + generate index.html |
| `bun run deploy:pages` | Build + deploy via gh-pages CLI |

### Post-Build Script

After Vite builds, `scripts/gen-index.mjs` automatically:
- Detects all client JavaScript and CSS files in `dist/client/assets/`
- Generates `dist/client/index.html` with proper asset references
- Applies the base URL from `VITE_BASE_URL` environment variable
- Adds correct meta tags for SEO and social sharing

## Configuration

### Base URL

- **GitHub Actions**: Uses `/pixel-sanctuary/` automatically
- **Manual deployment**: Use the `build:pages` script (already set to `/pixel-sanctuary/`)
- **Custom base**: `VITE_BASE_URL=/custom-path/ bun run build && node scripts/gen-index.mjs`

### Build Scripts in package.json

```json
{
  "build:pages": "VITE_BASE_URL=/pixel-sanctuary/ vite build && node scripts/gen-index.mjs",
  "deploy:pages": "npm run build:pages && gh-pages -d dist/client"
}
```

## Troubleshooting

### Deployment Workflow Takes Too Long

**Old Issue (Fixed):** The workflow was uploading assets but missing `index.html`
- **Solution**: Post-build script now generates it automatically
- Both build process and workflow have been updated

### App loads but shows blank page

**Issue**: Assets fail to load
- **Check**: Base URL configuration in `vite.config.ts`
- **Fix**: Ensure `VITE_BASE_URL=/pixel-sanctuary/` is set during build
- **Verify**: `dist/client/index.html` references assets with correct paths

### CSS/JS bundles fail to load

**Issue**: Asset references in `index.html` are incorrect
- **Check**: Run `bun run build:pages` locally and inspect `dist/client/index.html`
- **Verify**: All `<script>` and `<link>` tags point to `/pixel-sanctuary/assets/`
- **Fix**: The post-build script should handle this automatically

### Routes return 404 on page refresh

**Why**: GitHub Pages doesn't know about React Router routes
- **Status**: This is expected behavior for static hosting
- **Solution**: React Router handles routing client-side automatically
- **Note**: Deep links may need manual 404.html redirect (not currently needed)

### GitHub Actions keeps running

**Issue**: Missing `actions/configure-pages@v4` step
- **Solution**: Run workflow manually from Actions tab to reset
- **Check**: `.github/workflows/deploy.yml` has `Setup Pages` step
- **Status**: Fixed in updated workflow

## Environment Variables for CI/CD

If deploying elsewhere, set:
- `VITE_BASE_URL` - Base path for your deployment

Examples:

| Platform | Base URL | Command |
|----------|----------|---------|
| GitHub Pages | `/pixel-sanctuary/` | `VITE_BASE_URL=/pixel-sanctuary/ bun run build` |
| Cloudflare Pages | `/` | `bun run build` |
| Netlify | `/` | `bun run build` |
| Vercel | `/` | `bun run build` |

## Deployment from Merge Commits

If GitHub Actions workflow doesn't trigger on merge commits:
- Go to `.github/workflows/deploy.yml`
- Check `on.push.branches` includes `main`
- Manually trigger via "Run workflow" button in Actions tab
